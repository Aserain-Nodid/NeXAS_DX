# NeXAS_DX 使用指南

> ⚠️ 所有源代码必须始终保存为 UTF-8（无 BOM）并使用 CRLF 换行，这是唯一不可违背的规则。

## 快速开始
- 在 IDE 中运行 `com.giga.nexas.MainApplication`，或执行 `mvn javafx:run`。
- 窗口标题为 “NeXAS DX Converter”，程序会记住上一次的输入/输出目录以及输出锁定状态。

## 界面总览
- **顶部工具栏**：包含 `Settings` 菜单、`Engine` 下拉（BSDX / BHE）、`Always on Top` 开关；状态标签会自动换行显示扫描摘要，避免遮挡其它控件。
- **控制面板**：选择输入/输出目录（支持拖入目录）、锁定输出路径，可点击 `Run selected` 或 `Run all`。
- **进度提示栏**：进度条与状态标签固定在日志区右下角，实时展示批量与单文件任务进展。
- **左侧树视图**：按 “目录 → 类别 → 文件” 展示结构，二进制与 JSON 文件分组显示。
- **右侧卡片网格**：每张卡片代表一个类别，标题自动换行、列表可伸缩，并提供 `Parse all` / `Generate all` 快捷按钮。
- **底部日志面板**：输出所有操作结果（英文），右键支持复制或清空；右下角显示当前进度信息。

## 常见流程
1. **选择输入目录**：通过浏览按钮或拖拽目录到窗口，系统根据当前引擎收集相应扩展名。
2. **查看类别信息**：在树或卡片中浏览，二者会相互高亮；卡片提供数量统计与示例列表。
3. **执行操作**：
   - 双击树中的二进制/JSON 叶子节点，可立即解析或生成单个文件。
   - 点击 `Run selected` 执行当前选中的文件或类别。
   - 点击卡片上的 `Parse all` / `Generate all` 批量处理，或使用 `Run all` 执行全部类别。
4. **监控进度**：窗口右下角显示 “已完成 / 总数” 及当前文件，任务结束后状态标签会提示完成/失败并在日志中列出失败详情。
5. **检查输出**：日志记录生成文件的完整路径，输出会在选定的基目录下按原相对结构创建。

## 追加说明
- `Settings` 对话框可切换引擎和默认编码，变更会立即保存到 `Preferences` 并触发重新扫描。
- 日志及 UI 文本保持英文以便共享；代码注释可酌情使用简洁中文说明。
- 批量任务的失败条目会逐条附加到日志，便于排查。
- 请持续维护 `todolist.md`，记录后续优化或新功能。
